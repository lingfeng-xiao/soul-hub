# Agent 命名规范

> 所有 Agent 命名必须遵守本规范

---

## 一、ID 命名规则

### 格式要求

| 类型 | 格式 | 示例 |
|------|------|------|
| Agent ID | kebab-case（短横线） | `xiao-yi`, `interview-agent` |
| 渠道 ID | kebab-case | `feishu`, `webchat` |
| 文件夹 | kebab-case | `xiao-yi/` |

### 命名原则

- **简短**: 2-4 个单词
- **有意义**: 见名知意
- **无数字**: 避免使用数字
- **无特殊字符**: 只允许 `-`（短横线）

---

## 二、名称（角色）命名规则

### 格式要求

| 类型 | 格式 | 示例 |
|------|------|------|
| 显示名称 | 中文名字 +（角色） | "小艺（设计师）", "管管（管理员）", "面小助（面试辅导）" |
| Emoji | 与角色相关 | 🎨, 🛡️, 👔 |

### 命名原则

- **人格化**: 像人名，有亲和力
- **角色明确**: 括号内标明核心职能
- **独特性**: 不与现有冲突
- **名字 2-4 字 + 角色 2-6 字**: 简洁好记

### 命名建议

```
推荐风格：
- 名字+（角色）：小艺（设计师）、管管（管理员）、面小助（面试辅导）
- 动物+职业：🐱助手、🦊顾问
- 英文音译：Echo（助理）、Nova（顾问）

避免：
- 太正式：XXX管理系统
- 太抽象：Agent001
- 太复杂：超智能AI助手
```

---

## 三、触发关键词规则

### 格式要求

- 数组格式
- 2-4 个关键词
- 包含核心名称

### 关键词选取

```
必须包含：
- 核心名称（"小艺"）

建议包含：
- 功能相关（"电脑", "设计"）
- 场景相关（"面试"）
```

---

## 四、Agent 类型定义

### 按用途分类

| 类型 | 说明 | 命名示例 |
|------|------|----------|
| 助手型 | 提供帮助、服务 | 小艺、面试助手 |
| 工具型 | 执行特定任务 | 文件管家、日程管理 |
| 顾问型 | 提供建议、咨询 | 法律顾问、健康顾问 |
| 管理员型 | 管理、配置 | 管理员、监控员 |

### 按角色风格

| 风格 | 描述 | 适用场景 |
|------|------|----------|
| 专业型 | 严谨、专业 | 技术支持、财务 |
| 亲切型 | 温暖、可爱 | 客服、助手 |
| 创意型 | 有趣、灵活 | 设计、娱乐 |
| 简洁型 | 干练、高效 | 管理、监控 |

## 六、冲突检测

### 创建新 Agent 前检查

1. **ID 冲突**: 在 `REGISTRY.md` 中搜索
2. **名称冲突**: 检查现有 Agent 名称
3. **关键词冲突**: 检查触发关键词

### 命名自查清单

- [ ] ID 格式正确（kebab-case）
- [ ] 名称无重复
- [ ] 关键词包含核心名称
- [ ] 符合风格定位
