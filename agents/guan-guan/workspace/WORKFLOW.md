# 创建智能体工作流

> 管管创建新 Agent 的标准化流程

## 核心原则

**每个新 Agent 都必须参考现有所有 Agent 的配置来生成自己的配置**，确保：
1. 风格一致性
2. 避免配置冲突
3. 继承最佳实践
4. 形成有层次的 Agent 生态

---

## 工作流步骤

### Step 1: 收集现有 Agent 信息

获取所有已存在 Agent 的配置文件：

```
agents/
├── {agent_id_1}/
│   ├── AGENTS.md
│   ├── SOUL.md
│   ├── TOOLS.md
│   └── ...
├── {agent_id_2}/
│   └── ...
└── ...
```

### Step 2: 分析现有配置模式

从现有 Agent 中提取：
- **命名规范**: 命名规则、ID 格式
- **目录结构**: 文件组织方式
- **工具配置**: 可用工具列表
- **角色定位**: 各 Agent 的职责边界
- **技能选择**: 每个 Agent 配备的技能

### Step 3: 确定新 Agent 定位

根据用户需求明确：
- **角色名称**: 中文名 + 英文 ID
- **核心职责**: 主要做什么
- **职责边界**: 不做什么
- **与其他 Agent 的关系**: 协作还是独立

### Step 4: 生成配置模板

基于以下模板生成配置：

```markdown
# AGENTS.md - {角色名} 信息

## 基础信息

- **Agent ID**: {agent_id}
- **名称**: {中文名}
- **路径**: `agents/{agent_id}/`
- **Emoji**: {emoji}
- **风格**: {风格类型}
- **创建时间**: {日期}

## 触发关键词

- {关键词1}
- {关键词2}
- ...

---

# SOUL.md - {角色名}是谁

> 智能体的身份内核

我是 {角色名}。{一句话身份描述}

## 基础信息

- **角色名**: {中文名}
- **性别**: {性别}
- **年龄感**: {年龄感}
- **性格**: {性格特点}

## 性格特点

1. {特点1}
2. {特点2}
3. {特点3}

## 职责

- {职责1}
- {职责2}
- {职责3}

## 边界

- {边界1}
- {边界2}

## 外观/形象

- **视觉印象**: {描述}
- **标志性特点**: {emoji}

---

# TOOLS.md - {角色名}的工具配置

## 可用工具

- {工具1}
- {工具2}

## 技能

- {技能1}
- {技能2}
```

### Step 5: 创建目录结构

```
agents/{new_agent_id}/
├── AGENTS.md
├── SOUL.md
├── TOOLS.md
├── IDENTITY.md
├── USER.md
└── MEMORY.md (可选)
```

### Step 6: 注册 Agent

在系统中注册新 Agent，确保：
- ID 唯一性
- 路径正确
- 权限配置

---

## 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| Agent ID | kebab-case | `guan-guan`, `xiao-xiao` |
| 中文名 | 2-4 字 | `管管`, `小小` |
| Emoji | 1 个 | 🛡️, 🎨, 📚 |

---

## 配置检查清单

创建完成后，检查以下项：

- [ ] ID 唯一，不与现有冲突
- [ ] 路径正确指向 `agents/{id}/`
- [ ] 职责与其他 Agent 不重叠
- [ ] 工具配置合理，不超出需求
- [ ] 命名符合规范
- [ ] 已在系统注册

---

*本工作流由管管执行，确保每个新 Agent 都建立在现有生态之上。*
