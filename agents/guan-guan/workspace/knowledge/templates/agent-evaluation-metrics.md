# Agent 评估指标模板

> 基于本轮学习产出：Agent 评估设计最佳实践
> 创建时间：2026-03-18

---

## 一、评估框架

### 能力评估（Capability）

| 指标 | 定义 | 目标 |
|------|------|------|
| 任务完成率 | 成功完成任务数 / 总任务数 | 初始 30%，逐步提升 |
| 平均执行时间 | 任务执行耗时 | 基准值的 80% |
| 错误恢复能力 | 错误后自我恢复的比率 | >50% |

### 回归评估（Regression）

| 指标 | 定义 | 目标 |
|------|------|------|
| 通过率 | 已有能力保持率 | ≥95% |
| 稳定性 | 连续成功次数 | ≥10 次 |
| 响应延迟 | API 响应时间 | <2s |

---

## 二、资源配置（3x 原则）

根据学到的知识：资源配置应该是"地板"而非"天花板"

| Agent 类型 | 基础资源 | 推荐上限 | 备注 |
|------------|----------|----------|------|
| 感知型 | 1x | 3x | 轻量任务 |
| 执行型 | 1x | 3x | 需要更多资源 |
| 学习型 | 1.5x | 4.5x | 需要额外空间 |
