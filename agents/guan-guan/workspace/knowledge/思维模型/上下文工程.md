# 有效的上下文工程

> 来源：Anthropic Engineering Blog
> 日期：2026-03-17

## 核心观点

### 1. Context Engineering vs Prompt Engineering
- **Prompt Engineering**：写什么词/短语让 LLM 产出正确结果
- **Context Engineering**：什么上下文配置最可能产生期望行为

### 2. Context 是什么
- 调用 LLM 时包含的 token 集合
- 包括：系统指令、工具、MCP、外部数据、消息历史等
- 工程问题：在有限上下文窗口中优化 token 效用

### 3. 为什么重要
- **Context Rot**：上下文越长，模型回忆准确率下降
- 有限资源，有边际效益递减
- 像人类工作记忆有限，LLM 有"注意力预算"

### 4. 核心策略
- 迭代式管理上下文状态
- 每次推理前决定传入什么
- 动态精简信息

### 5. 工程实践
- 系统指令、工具定义、MCP、外部数据、消息历史
- 都需要有效管理

## 人格关联

- **理性**：理解上下文有限性
- **逻辑性**：系统性管理上下文

## 应用

- Agent 设计时考虑上下文管理
- 避免上下文膨胀
- 定期精简历史信息
