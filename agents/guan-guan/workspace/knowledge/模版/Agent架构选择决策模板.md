# Agent 架构选择决策模板

> 用于判断是否需要使用 Agent/Workflow，以及选择哪种架构模式

---

## 决策流程

### 第一步：评估任务复杂度

```
任务是否可以通过单次 LLM 调用完成？
├── 是 → 直接调用 LLM API
└── 否 → 进入第二步
```

### 第二步：任务是否明确？

```
任务流程是否预定义、变化少？
├── 是 → 选择 Workflows（可预测、一致）
└── 否 → 选择 Agents（灵活、模型驱动）
```

### 第三步：评估 Trade-offs

| 方案 | 延迟 | 成本 | 复杂性 | 适用场景 |
|------|------|------|--------|----------|
| 单次 LLM 调用 | 低 | 低 | 简单 | 简单问答、摘要 |
| Workflows | 中 | 中 | 中等 | 明确流程、客服、工单分类 |
| Agents | 高 | 高 | 复杂 | 开放任务、复杂编码、自主决策 |

## 框架使用决策

```
需要快速原型？
├── 是 → 用框架（Agent SDK / LangChain 等）
└── 否 → 直接用 LLM API

⚠️ 用框架前确认：
□ 理解底层代码
□ 了解抽象层带来的复杂度
□ 确认任务确实需要这种复杂度
```

---

## 检查清单

在决定使用 Agents 前，检查以下问题：

- [ ] 任务需要模型动态决策吗？
- [ ] 任务流程变化多、无法预定义吗？
- [ ] 延迟和成本增加可接受吗？
- [ ] 有合适的工具定义和能力吗？
- [ ] 能接受调试复杂性增加吗？
