# 测试知识

> 定时任务工作流测试经验总结

## 测试金字塔

```
        /\
       /  \    验收测试 (E2E)
      /----\   系统测试
     /      \  集成测试
    /--------\ 单元测试
```

| 类型 | 目的 | 数量 |
|------|------|------|
| 单元测试 | 验证单个组件/功能 | 多 |
| 集成测试 | 验证组件间交互 | 中 |
| 系统测试 | 验证完整工作流 | 少 |
| 验收测试 | 验证业务需求 | 最少 |

---

## 测试用例设计原则

### 验证点清单

每个测试用例必须包含明确的验证点：

1. **前置条件** - 测试开始前的状态
2. **测试步骤** - 具体的操作
3. **验证点** - 需要验证的结果
4. **预期结果** - 期望的结果
5. **实际结果** - 实际发生的情况
6. **通过/失败** - 是否符合预期

### 执行记录

每次测试必须记录：
- 测试时间
- 测试者
- 测试结果
- 发现的问题

## 定时任务工作流测试用例

### ST-01: 完整工作流

| 步骤 | 验证点 | 预期结果 |
|------|--------|----------|
| 1 | 申请消息发送 | 消息成功发送到群 |
| 2 | 审批消息发送 | 审批消息成功发送 |
| 3 | cron 任务创建 | 任务出现在 cron list |
| 4 | 上线通知发送 | 上线通知成功发送 |
| 5 | 审计日志 | 日志包含新记录 |
| 6 | 清理会话 | 临时会话被清理 |
| 7 | 删除测试任务 | 测试任务被删除 |

## 问题记录

### 问题1：审计日志未自动记录

**现象**：创建任务后，审计日志为空

**原因**：创建任务时没有自动记录审计日志

**解决方案**：
- 在 cron-manager skill 中明确步骤
- 强制要求先记录日志再创建任务
- 完成后手动补充审计记录

### 问题2：临时会话未清理

**现象**：工作流执行后，subagent 会话仍然存在

**解决方案**：
- 执行完成后使用 subagents list 查看
- 清理 done 状态的会话
- 在 skill 中增加清理步骤

### 问题3：测试任务未删除

**现象**：测试完成后，测试任务仍在 cron list 中

**解决方案**：
- 测试完成后立即删除测试任务
- 在 skill 中增加删除步骤

## 教训

1. **测试必须有明确的验证点**
2. **测试后必须验证结果**
3. **发现问题要立即修复**
4. **知识要及时沉淀**
5. **每次学习新知识后更新知识库**
