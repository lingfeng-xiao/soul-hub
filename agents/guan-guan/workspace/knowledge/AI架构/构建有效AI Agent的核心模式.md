# 构建有效 AI Agent 的核心模式

> 来源: Anthropic Engineering Blog (2024-12-19)
> 主题: Agent 架构设计

## 核心观点

**最成功的 Agent 实现使用简单、可组合的模式，而非复杂框架。**

## 关键架构区分

| 类型 | 定义 | 适用场景 |
|------|------|----------|
| **Workflows** | 通过预定义代码路径编排 LLM 和工具 | 任务可预测、需要一致性 |
| **Agents** | LLM 动态指导自己的流程和工具使用 | 需要灵活性、模型驱动决策 |

## 推荐工作流模式

### 1. Prompt Chaining（提示链）
- 将任务分解为序列步骤，每个 LLM 调用处理前一个的输出
- 适用：任务可清晰分解为固定子任务
- 示例：写营销文案 → 翻译成其他语言

### 2. Routing（路由）
- 分类输入并导向专门的 followup 任务
- 适用：复杂任务有不同类别，需要分离处理
- 示例：客服查询分类、技术支持 vs 退款

### 3. Parallelization（并行化）
- LLM 同时处理子任务，输出聚合
- 适用：独立子任务、需要多样性
- 示例：摘要多个文档、投票选择最佳方案

## 何时使用 Agent

**原则：先找最简单的解决方案，只在需要时增加复杂度。**

- 单次 LLM 调用 + 检索 + 上下文示例 = 多数应用的最优解
- Agent 交换延迟和成本换取更好的任务性能
- 需要考虑这个权衡是否值得

## 框架使用建议

可用框架：Claude Agent SDK、AWS Strands、Rivet、Vellum

**建议**：
- 先直接使用 LLM API，许多模式几行代码就能实现
- 使用框架时，确保理解底层代码
- 错误假设是常见错误来源
