# 长时运行 Agent 框架

> 来源：Anthropic Engineering Blog
> 日期：2026-03-17

## 核心观点

### 1. 长时运行问题
- Agent 需要跨多上下文窗口工作
- 每次新会话没有之前记忆
- 类似工程师换班，每班不知道上发生了什么

### 2. 两层解决方案

**Initializer Agent**
- 首次运行时设置环境
- 一次性初始化

**Coding Agent**
- 每次会话增量推进
- 留下清晰文档给下一会话

### 3. 失败模式

**模式1：一次做太多**
- 试图一次性完成
- 中间耗尽上下文
- 下次从半成品开始

**模式2：后续迷失**
- 获得进展后迷失方向
- 难以保持连贯性

### 4. 关键技巧
- 增量推进
- 清晰交接文档
- compaction 不完美

## 应用
