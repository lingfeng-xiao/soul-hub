# Anthropic 构建有效 AI Agents 核心原则

> 来源: https://www.anthropic.com/engineering/building-effective-agents
> 日期: 2026-03-18

## 核心观点

最成功的实现使用的是简单的、可组合的模式，而不是复杂的框架。

## 关键原则

### 1. 简单优先 (Simple First)

- 找到最简单的解决方案
- 只在需要时才增加复杂性
- 优化单个 LLM 调用 + 检索 + 上下文示例 通常就足够了

### 2. 工作流 vs Agent

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| **Workflows** | 预定义代码路径 | 任务明确、需要可预测性 |
| **Agents** | LLM 动态控制 | 需要灵活性、模型驱动决策 |

### 3. 何时使用框架

- 框架简化了标准低级任务：调用 LLM、定义/解析工具、链式调用
- 但会增加抽象层，掩盖底层提示和响应
- **建议**：先用 LLM API 直接实现，很多模式只需几行代码

## 5 种工作流模式

### 1. Prompt Chaining (链式)

将任务分解为步骤序列，每个 LLM 处理前一个的输出。

**适用场景**：
- 营销文案生成 → 翻译
- 大纲生成 → 检查 → 写文档

### 2. Routing (路由)

分类输入并引导到专门的后续任务。

**适用场景**：
- 客服分类（一般问题/退款/技术支持）
- 简单问题 → 小模型，复杂问题 → 大模型

### 3. Parallelization (并行化)

LLM 同时处理任务，程序化聚合输出。

**两种形式**：
- Sectioning: 分解为独立子任务并行
- Voting: 多次运行获取多样化输出

**适用场景**：
- Guardrails：一个模型处理查询，另一个筛选不当内容
- 代码审查：多个提示审查漏洞

### 4. Orchestrator-workers (编排器-工作者)

中央 LLM 动态分解任务，委托给工作者 LLM，并综合结果。

**适用场景**：
- 复杂的多文件代码修改
- 跨多源的信息搜索和分析

### 5. Evaluator-optimizer (评估器-优化器)

一个 LLM 生成响应，另一个提供评估和反馈，循环迭代。

**适用场景**：
- 文学翻译（需要迭代优化）
- 复杂搜索任务（多轮搜索和分析）

## Agent 的关键能力

Agent 在生产中兴起得益于 LLM 在以下方面的成熟：

1. **理解复杂输入**
2. **推理和规划**
3. **可靠使用工具**
4. **从错误中恢复**

## 核心要点总结

1. **不要过早复杂化** - 从简单开始
2. **选择合适的模式** - 根据任务性质选择工作流
3. **理解底层原理** - 即使使用框架
4. **关注实际效果** - 延迟和成本的权衡
