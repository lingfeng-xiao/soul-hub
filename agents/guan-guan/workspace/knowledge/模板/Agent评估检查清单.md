# Agent 评估检查清单模板

> 用于快速构建 Agent 评估系统的清单

## 一、评估目标定义

- [ ] **核心任务**：Agent 需要完成什么？
- [ ] **成功标准**：如何定义"成功"？
- [ ] **失败场景**：哪些情况算失败？

## 二、任务设计

- [ ] **输入定义**：任务的输入是什么？
- [ ] **环境要求**：需要什么工具/资源？
- [ ] **边界条件**：边界案例有哪些？

## 三、评分器设计

- [ ] **自动评分**：哪些可以自动验证？
- [ ] **LLM 评分**：哪些需要 LLM 评判？
- [ ] **人工校准**：多久需要人工检查？

## 四、试次配置

- [ ] **试次数量**：至少 10 次？
- [ ] **随机种子**：是否固定？
- [ ] **超时设置**：每个任务多久？

## 五、资源配置

- [ ] **CPU 限制**：分配多少？
- [ ] **内存限制**：分配多少？
- [ ] **头room设置**：是否需要缓冲？

## 六、执行计划

```
第1步：定义 3-5 个核心任务
第2步：为每个任务设计成功标准
第3步：实现自动评分器
第4步：运行 10+ 次试次
第5步：分析结果，识别问题
第6步：迭代改进
```

---

**使用示例**：

为一个客服 Agent 创建评估：

1. **核心任务**：回答用户问题
2. **成功标准**：
   - 正确性（答案准确）
   - 完整性（覆盖问题要点）
   - 礼貌性（语气友好）
3. **任务示例**：
   - 退换货请求
   - 产品咨询
   - 技术问题
4. **评分器**：
   - 自动：答案包含关键字？
   - LLM：答案是否有帮助？
