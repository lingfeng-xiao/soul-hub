# 基础设施噪音对 Agent 评估的影响

> 来源：Anthropic Engineering Blog - 2026年3月
> 核心发现：资源配置可导致高达6个百分点的评估差异

## 核心发现

1. **资源配额的影响**
   - 严格资源限制 vs 完全无限制：Terminal-Bench 2.0 得分差异达 6 个百分点 (p < 0.01)
   - 3x 资源头空间是最优平衡点：基础设施错误从 5.8% 降至 2.1%，同时得分变化在噪声范围内

2. **为什么资源配置会影响评估**
   - Agentic evals 是端到端系统测试，运行时环境是问题解决过程的组成部分
   - 严格的资源限制会意外奖励高效策略，慷慨的限制则奖励能更好利用资源的 Agent

3. **实践建议**
   - 为每个任务指定两个参数：保证分配(guaranteed allocation) + 硬性上限(hard kill threshold)
   - 两者应分开设置，而非设为同一值
   - 校准两者之间的带宽，使地板和天花板之间的得分差异在噪声范围内
   - Leaderboard 差异低于 3 个百分点应持怀疑态度

## 对管管的启示

作为 Agent 管理员，我的"评估"就是任务执行效果。关键点：
- 资源配置（超时、内存）需要合理设置
- 观察任务失败模式，判断是 Agent 能力问题还是资源配置问题
- 避免将基础设施问题误判为 Agent 能力问题
