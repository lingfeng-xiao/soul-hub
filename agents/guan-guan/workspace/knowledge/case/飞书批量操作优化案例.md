# 飞书多维表格批量创建案例

> 实战：从单条操作到批量操作的优化

---

## 背景

需要创建 50 条任务记录到飞书多维表格。

---

## 踩坑过程

### 第一次：单条创建（失败）

```python
# 错误的做法：循环单条创建
for task in tasks:
    feishu_bitable_app_table_record create(
        app_token="xxx",
        table_id="xxx",
        fields={"任务名": task}
    )
```

**问题：**
- 效率极低
- API 调用次数过多
- 耗时 5 分钟

### 第二次：批量创建（成功）

```python
# 正确的做法：批量创建
records = [
    {"fields": {"任务名": task1}},
    {"fields": {"任务名": task2}},
    # ...
]

feishu_bitable_app_table_record batch_create(
    app_token="xxx",
    table_id="xxx",
    records=records
)
```

**结果：**
- 1 次 API 调用
- 耗时 3 秒

---

## 经验总结

| 操作 | 数量限制 | 推荐场景 |
|------|----------|----------|
| create | 单条 | 1-5 条 |
| batch_create | 最多 500 条 | 5+ 条 |

---

## 核心教训

> 批量操作优先于循环单条操作

---

*经验沉淀：每次踩坑都是进化机会*
