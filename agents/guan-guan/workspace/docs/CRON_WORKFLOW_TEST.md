# 定时任务创建工作流测试方案

> 测试方案设计原则：先设计测试，再实现功能

---

## 一、测试策略

### 测试金字塔

```
        /\
       /  \    验收测试 (E2E)
      /----\   系统测试
     /      \  集成测试
    /--------\ 单元测试
```

### 测试类型

| 类型 | 目的 | 数量 |
|------|------|------|
| 单元测试 | 验证单个组件/功能 | 多 |
| 集成测试 | 验证组件间交互 | 中 |
| 系统测试 | 验证完整工作流 | 少 |
| 验收测试 | 验证业务需求 | 最少 |

## 三、测试用例详情

### ST-01: 完整工作流

**前置条件**：
- 共享规则存在
- cron-manager skill 存在

**测试步骤**：
1. 小艺发送申请消息（message 工具）
2. 管管审批通过
3. 管管创建 cron 任务
4. 小艺发送上线通知

**验证点**：
- [ ] 申请消息出现在群里
- [ ] 审批消息出现在群里
- [ ] cron list 包含新任务
- [ ] 上线通知出现在群里
- [ ] 审计日志有新记录

### ST-02: 审批拒绝

**测试步骤**：
1. 小艺发送申请（配置有问题）
2. 管管审批拒绝
3. 检查 cron list 无新任务

**验证点**：
- [ ] 拒绝消息出现在群里
- [ ] cron list 无新任务
- [ ] 无审计日志记录
