# 基础设施噪声对 Agent 评估的影响

**来源**: Anthropic Engineering Blog  
**日期**: 2026-03-18  
**价值分**: 4/5

## 核心要点

1. **资源配额 = 评估变量**
   - 基础设施配置可导致 6% 的分数差异（Terminal-Bench 2.0）
   - 有时超过顶级模型之间的排行榜差距

2. **3x 阈值是关键转折点**
   - <3x: 主要修复基础设施可靠性（瞬时资源峰值）
   - >3x: 真正改变评估内容，让 agent 能用更多资源解决问题

3. **资源限制改变评估本质**
   - 严格限制 → 奖励 lean 高效策略
   - 宽松限制 → 奖励 brute-force 资源利用能力

## 应用到管管系统

| 场景 | 当前配置 | 优化建议 |
|------|---------|---------|
| Cron 任务超时 | 45s/120s | 考虑任务复杂度，给复杂任务 3x 基础时间 |
| Agent 并发数 | 1 | 可尝试小规模并发测试 |
| 内存/资源限制 | 无明确限制 | 设定明确阈值，避免"宽松即高分" |

## 模板：Cron 任务资源评估

```
## 任务: {name}
- 当前超时: {timeout}s
- 复杂度: 低/中/高
- 建议超时: {base} * {multiplier}x
```
