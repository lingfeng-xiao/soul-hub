# 基础设施噪声对 Agent 评估的影响

## 核心发现
- **资源配置可导致高达 6 个百分点的差异**，有时超过模型之间的 leaderboard 差距
- Agentic coding evals 与静态 benchmarks 不同：运行时环境是问题解决过程的一部分

## 关键数据
| 配置 | Infra 错误率 | 成功率变化 |
|------|-------------|------------|
| 1x (严格) | 5.8% | 基准 |
| 3x 头部空间 | 2.1% | 噪声范围内 |
| 无限制 | 0.5% | +6个百分点 |

## 可行动要点

### 1. 资源配置要分层
- **Guaranteed allocation**: 预留给容器的资源
- **Hard limit**: 超过则 kill 容器
- 推荐设置：**3x ceiling** 作为合理平衡点

### 2. 资源限制不是中性的
- 严格限制 → 奖励高效、简洁的代码策略
- 慷慨限制 → 奖励能充分利用资源的 agent
- 不同资源策略 = 不同能力的测量

### 3. 评估建议
- 指定资源时同时说明 enforcement methodology
- 多时间点运行取平均，减少时间相关噪声
- 将资源配置视为第一类实验变量

## 对管管的启示
在设计 Agent 评估/心跳任务时，需要考虑资源配置的影响，避免将基础设施差异误认为能力差异。
