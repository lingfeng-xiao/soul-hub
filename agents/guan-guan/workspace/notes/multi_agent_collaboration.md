# 多 Agent 协作系统设计笔记

## 核心要点
多 Agent 协作系统类似于 MCP 协议在 Agent 层面的扩展，核心是**通信协议 + 职责分离**。

## 设计模式

### 1. 主从架构 (Leader-Follower)
- Leader Agent 负责任务分发和结果汇总
- Follower Agent 执行具体子任务
- 适用场景：任务可清晰拆分

### 2. 对等架构 (Peer-to-Peer)
- 所有 Agent 平等协作
- 通过消息队列或事件总线通信
- 适用场景：复杂推理、多视角分析

### 3. 层级架构 (Hierarchical)
- 多层 Agent，每层负责不同抽象级别
- 类似于公司组织架构
- 适用场景：大规模系统

## 通信协议设计
参考 MCP 的三种能力：
1. **Tools**: Agent 能力暴露
2. **Resources**: 共享上下文/状态
3. **Prompts**: 标准化任务模板

## 协作模式
- **串行**: A → B → C，依赖链
- **并行**: A → [B, C, D]，广播
- **循环**: A ↔ B ↔ C，反馈迭代

## 可应用到管管的点
1. **子 Agent 通信**: 建立标准消息格式
2. **任务分发**: 参考主从架构
3. **状态共享**: 使用共享内存/文件
4. **监控协作**: 用 cron 定时检查子 Agent 状态
