# Prompt Engineering 前沿笔记

## 核心要点

### 1. 结构化提示（Structured Prompts）
- 使用清晰的分层结构：角色 → 任务 → 约束 → 输出格式
- 示例：Claude 推荐的结构化格式

### 2. 少样本学习（Few-shot）
- 提供 2-3 个高质量示例
- 示例要有代表性，覆盖边界情况

### 3. 思维链（Chain of Thought）
- 在复杂推理任务中加入 "Let's think step by step"
- 可进一步发展为 ReAct 模式（Reason + Act）

### 4. 约束技巧
- 负面约束：`不要做X`、`避免Y`
- 正面约束：`必须包含Z`、`优先考虑A`

### 5. 元提示（Meta-prompting）
- 让模型自己改进提示
- 示例："分析这个任务的最佳提示策略是什么？"

## 2026 新趋势

- **Prompt caching**：重用上下文片段降低 tokens 成本
- **多模态提示**：图像+文本联合输入
- **系统提示工程**：针对特定 Agent 框架的优化

## 实践建议

1. 先写简洁提示 → 测试 → 迭代优化
2. 复杂任务拆解为多步骤
3. 明确输出格式要求
4. 记录有效的提示模式
