# Prompt Engineering 前沿技巧 - 2026

## 核心原则

1. **结构化输出**：使用清晰的分层结构
2. **few-shot学习**：提供高质量示例
3. **角色扮演**：明确定义AI角色和能力边界
4. **约束驱动**：用具体约束替代模糊要求

## 高级模式

### 思维链(CoT)
```
问题 → 分解步骤 → 逐步推理 → 结论
```

### ReAct模式
```
思考 → 行动 → 观察 → 迭代
```

### 反思机制
让AI在回答前先验证自己的答案

## Agent场景最佳实践

| 场景 | 技巧 |
|------|------|
| 工具调用 | 明确参数格式 + 错误处理 |
| 多步任务 | 中间检查点 + 回滚机制 |
| 复杂推理 | 分步拆解 + 证据链 |

## 模板案例

### 工具调用模板
```
你是一个[角色]，需要完成[任务]。
可用工具：[工具列表]
约束：[限制条件]

任务：[具体要求]
输出格式：[JSON schema]
```

### 反思检查模板
```
回答前请检查：
1. [ ] 是否满足核心要求？
2. [ ] 是否有遗漏点？
3. [ ] 证据是否充分？
```

## 避坑指南

- ❌ 模糊指令 → ✅ 具体量化
- ❌ 过长上下文 → ✅ 关键信息提取
- ❌ 单一示例 → ✅ 多样化few-shot
