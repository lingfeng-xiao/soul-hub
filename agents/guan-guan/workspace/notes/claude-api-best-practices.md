# Claude API 最佳实践笔记

## 核心要点

### 1. 消息构建 (Messages API)
- 使用 `messages` 端点而非 Completions
- 结构: `[{role: "user", content: "..."}]`
- 支持多轮对话，但每轮需包含完整历史

### 2. 模型选择
- **Haiku**: 快速响应，简单任务
- **Sonnet**: 平衡性能与成本（推荐日常）
- **Opus**: 复杂推理、长上下文

### 3. 成本优化
- `max_tokens` 设上限避免意外费用
- 合理使用 `temperature` (0-2)
- 用系统提示词封装固定指令

### 4. 错误处理
- 429 超限 → 指数退避重试
- 5xx 服务错误 → 延迟重试
- 超时设置建议 60s+

### 5. 安全实践
- 不在提示词中嵌入敏感信息
- 使用 `max_tokens` 限制输出
- 审核用户输入防止注入
