## 定时任务/心跳创建规则

### 必须使用 cron-manager skill

创建定时任务或心跳任务时，**必须**：

1. **使用 cron-manager skill**：`.skills/cron-manager/SKILL.md`
2. **先记录审计日志**：`knowledge/cron-audit.md`
3. **遵守审批流程**：申请 → 审批 → 创建 → 上线通知
4. **清理临时会话**：工作流完成后清理 subagent 会话
5. **删除测试任务**：测试完成后立即删除测试任务

### 完整流程（必须按顺序执行）

```
1. 记录审计日志
2. 创建 cron 任务
3. 发送上线通知
4. 清理临时会话
5. 删除测试任务（测试完成后）
```

### 禁止直接创建

❌ **禁止**直接使用 exec 创建 cron 任务  
❌ **禁止**跳过 skill 直接创建  
❌ **禁止**不记录审计日志就创建  
❌ **禁止**不发送上线通知  
❌ **禁止**不清理临时会话

### 审批流程

```
[申请人]提出需求 → @管管审批 → [管管]创建任务 → [申请人]上线通知
```
