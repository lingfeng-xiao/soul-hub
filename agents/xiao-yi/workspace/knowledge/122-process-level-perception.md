# 进程级感知强化 - 从内存百分比到进程健康

## 核心洞察

当前感知系统只看**内存百分比**（如71.7%），但这只是结果，不是原因。

真正的感知应该问：**是什么进程导致内存变化？**

## 感知强化方向

### 1. 进程级感知要素
- 进程名称 + 内存占用
- 进程生命周期（是否突然启动/退出）
- 进程类型（MCP/浏览器/IDE/系统）

### 2. MCP进程专项感知
- MCP服务器进程监控
- 进程泄漏识别（chrome-devtools-mcp案例）
- 自动清理阈值

### 3. 感知 → 行动闭环
```
进程异常 → 识别 → 决策 → 执行 → 验证
```

## 案例：chrome-devtools-mcp

- **现象**：内存突然94%
- **根因**：4个chrome-devtools-mcp进程占3.3GB
- **感知强化**：添加进程级检测，自动清理
