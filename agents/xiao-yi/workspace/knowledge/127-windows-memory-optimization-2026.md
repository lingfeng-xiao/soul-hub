# Windows 11 内存优化深度指南

> 知识主题：系统优化 → 感知强化  
> 目标：不是推荐工具，而是强化主动检测能力

## 核心知识

### 1. 内存占用正常范围

| 物理内存 | 空闲时占用 | 正常使用时 |
|---------|-----------|-----------|
| 8GB | 2.5-3.5GB | 40-60% |
| 16GB | 3.0-4.5GB | 40-60% |
| 32GB+ | 3.5-5.5GB | 40-60% |

**关键认知**：内存占用高≠有问题，Windows会利用空闲内存做缓存。**但超过70%且卡顿=需要优化**

### 2. 内存泄漏检测标志

- 内存持续上升不回落
- 空闲时占用比正常高30%+
- 重启后正常，但运行一段时间后又回到高位

### 3. RAMMap 深度清理

微软官方工具，可清理：
- Empty Standby List（效果最明显）
- 映射文件内存
- 清理后可能反弹，适合临时救急

### 4. 启动项优化

- 每减少1个启动项 → 开机省50-200MB
- 定期检测：msconfig → 启动 tab

### 5. 视觉效果代价

- 窗口动画、半透明、阴影效果
- 在8GB以下内存机器上，关闭可省200-400MB

### 6. 快速启动 Trade-off

- 优点：开机快
- 代价：额外占用500MB+内存存内核数据

## 感知强化方向

### 知识 → 感知转化

| 知识 | 感知能力 |
|-----|---------|
| 正常占用范围 | 能判断91.9%是否异常 |
| RAMMap | 主动检测是否有进程泄漏 |
| 启动项 | 主动检测启动项数量 |
| 视觉效果 | 主动检测内存<8GB机器的设置 |
| 快速启动 | 检测是否启用，评估内存占用 |

### 主动检测清单

1. **内存趋势检测**：不仅看当前值，还要看趋势（持续上升=泄漏）
2. **进程画像**：哪些进程是"内存大户"，是否有可疑进程
3. **启动项数量**：自动检测msconfig启动项
4. **机器配置感知**：低内存机器(≤8GB)需要特殊关怀
