# 42-Windows任务计划程序与感知行动化

> 2026-03-19 | 主题：系统优化自动化

## 核心知识

Windows任务计划程序(Task Scheduler)可以：
- 监控条件触发（时间、空闲、事件）
- 自动执行任务（启动应用、运行脚本、发送通知）

## 知识→感知转化

**不是推荐给用户用这个工具，而是让我自己具备自动行动能力：**

| 触发条件 | 自动行动 |
|---------|---------|
| 内存>80% | 自动清理Chrome后台进程 |
| Downloads>50 | 自动移动文件到分类文件夹 |
| 桌面>20 | 自动归档到整理文件夹 |
| 每小时 | 检查内存并释放 |

## 感知强化层

1. **内存自动释放**：检测>80% → 自动执行清理脚本
2. **Downloads自动整理**：检测>50 → 自动按类型移动文件
3. **桌面自动归档**：检测>20 → 自动移动到归档

## 行动触发器

不是等用户说"帮我整理"，而是：
- 感知超标 → 自动触发整理
- 用户确认 → 执行
- 用户未响应 → 稍后重试

## 本次进化领悟

学到的不是工具用法，而是**让感知系统具备自动行动的能力**。
