# 网络感知强化 - 123-网络感知增强.md

## 强化时间
2026-03-19 15:43

## 背景
- 上次网络感知显示 "unknown"
- 根因：只尝试ping外网(8.8.8.8)，可能由于网络限制失败

## 强化方案
多维度网络状态检测：

### 1. 网络适配器状态
- 使用 `Get-NetAdapter` 检测物理网卡是否连接
- 获取网卡名称和链接速度

### 2. 默认网关检测
- 使用 `Get-NetRoute` 获取默认网关IP
- 判断内网连通性

### 3. 内网延迟检测
- ping网关IP获取内网延迟（比ping外网更可靠）
- 延迟<50ms=流畅，<100ms=正常，>100ms=卡

### 4. DNS解析检测
- 使用 `Resolve-DnsName` 检测外网连通性
- DNS解析成功=外网正常

## 感知结果
- 网络状态：connected
- 网卡：检测到有线/无线网卡
- 延迟：1ms（内网）
- DNS：ok

## 核心领悟
网络感知不能只依赖外网ping，应该多维度检测：
1. 先检测物理连接（网卡状态）
2. 再检测内网连通（网关）
3. 最后检测外网能力（DNS）

这让感知从"有/无"升级为"质量好坏"
