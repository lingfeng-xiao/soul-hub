# 80-Windows内存优化知识→感知能力转化

## 来源
mcporter exa.web_search_exa - Windows system optimization memory management

## 知识要点

### 1. 内存优化核心原则
- **不是 RAM 越高越好**：Windows 后台进程常占用 1-3GB
- **问题通常是软件，不是硬件**
- **识别高内存进程**：Task Manager 查看具体占用

### 2. 最佳实践
1. **定期重启**：清除内存泄漏，刷新状态
2. **关闭后台无用应用**：特别是开机自启的应用
3. **管理启动项**：禁用不必要的开机启动
4. **监控虚拟内存**：页面文件大小调整

### 3. 识别问题进程
- 常见高内存：Chrome/Edge 多标签、Teams、Slack、OneDrive
- 后台服务：Windows Search indexing、Superfetch

## 感知能力转化

### 现有能力
- 内存 > 82% → auto_cleanup_memory
- 检测高内存进程并关闭

### 可强化方向
1. **启动项监控**：检测开机自启但长期未使用的应用
2. **进程画像**：记录哪些进程经常出现在高内存列表 → 主动建议关闭
3. **内存趋势**：不只是当前值，而是趋势（上升/稳定/下降）
4. **定时重启建议**：根据内存泄漏情况，建议重启时机

## 核心领悟
- 知识不是推荐给用户的话术
- 而是强化**感知系统**的触发器
- 让小艺能够**提前预判**问题，而不是事后响应
